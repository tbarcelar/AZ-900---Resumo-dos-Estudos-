# AZ 900 - Resumo dos Estudos

**Autenticação**
 estabelece a identidade do usuário, mas a autorização é o processo de estabelecer o nível de acesso de uma pessoa ou serviço autenticado. Ele especifica quais dados eles têm permissão para acessar e o que podem fazer com eles.

 **O acesso condicional**
 permite que você exija que os usuários acessem seus aplicativos apenas de dispositivos aprovados ou gerenciados.


**Azure Active Directory**
 fornece serviços de identidade que permitem que seus usuários entrem e acessem os aplicativos em nuvem da Microsoft e os aplicativos em nuvem que desenvolve. 
 Se já trabalhou com o Active Directory, o Azure AD será familiar, o Azure AD pode detectar tentativas de entrada em locais inesperados ou dispositivos desconhecidos.

**O Azure AD Connect**
sincroniza identidades de usuário entre o Active Directory em execução local e o Azure AD. O Azure AD Connect sincroniza alterações entre os dois sistemas de identidade, o que permite que você use recursos como SSO, MFA e redefinição de senha de autoatendimento em ambos os sistemas. A redefinição de senha de autoatendimento evita que os usuários usem senhas comprometidas conhecidas.

**autenticação multifator (MFA)**
é um processo em que um usuário é solicitado durante o processo de login para uma forma adicional de identificação, como inserir um código em seu telefone celular ou fornecer uma leitura de impressão digital,
biométricas, como reconhecimento facial.

**O SSO**
 permite que um usuário lembre-se de apenas um ID e uma senha para acessar vários aplicativos


**RBAC** 
 O RBAC do Azure permite que você crie funções que definem as permissões de acesso. Você pode criar uma função que limite o acesso apenas às máquinas virtuais e uma segunda função que forneça aos administradores acesso a tudo.

 **As tags**
  fornecem informações extras, ou metadados, sobre seus recursos. Eles podem criar uma tag chamada BillingDept cujo valor seria o nome do departamento de cobrança. Você pode usar a Política do Azure para garantir que as marcas adequadas sejam atribuídas quando os recursos são provisionados. Assim é possivel saber o faturamente de cada setor. 

  **Trust Center**
   é um ótimo recurso para pessoas em organização que podem desempenhar um papel na segurança, privacidade e conformidade. Onde uma equipe jurídica pode acessar informações sobre como a nuvem da Microsoft os ajuda a proteger dados confidenciais e cumprir as leis e regulamentos aplicáveis

**Blueprints**
Em vez de ter que configurar recursos como a Política do Azure para cada nova assinatura, Blueprints do Azurepermite que você defina um conjunto repetível de ferramentas de governança e recursos padrão do Azure que sua organização requer. Isso permite que as equipes de desenvolvimento criem e implantem rapidamente novos ambientes com o conhecimento de que estão construindo dentro da conformidade organizacional com um conjunto de componentes integrados que aceleram as fases de desenvolvimento e implantação.
